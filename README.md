# Olist Marketplace ETL & Business Intelligence Project

## Project Overview

This project demonstrates the complete design and implementation of an **ETL (Extract, Transform, Load) pipeline** and a **Business Intelligence solution** using the **Olist Brazilian E-commerce Dataset**.

The objective was to build a **SQL Server data warehouse** capable of answering key business questions about marketplace sales performance, customer behavior, product categories, logistics, and seller performance.

The project covers every stage of the analytics lifecycle:

- Data Profiling
- Data Warehouse Modeling
- Data Cleaning and Transformation
- ETL Pipeline Implementation
- Data Validation
- Business KPI Development
- Interactive Power BI Dashboard

---

## Business Questions

This project answers the following business questions:

- What is the total revenue generated by the marketplace?
- How have sales changed over time?
- Which product categories generate the highest revenue?
- Which product categories receive the most orders?
- Which customer states generate the most sales?
- Which sellers generate the highest revenue?
- What is the average delivery time?
- How many orders are delivered late?

---

## Data Profiling

Before building the data warehouse, an extensive data profiling process was performed to evaluate the quality of the source data.

The analysis included:

- Missing values
- Duplicate records
- Invalid values
- Negative and zero values
- Inconsistent text formatting
- Leading and trailing spaces
- Referential integrity checks
- Outlier detection
- Date consistency validation
- Category normalization
- Delivery time analysis

This step ensured that potential data quality issues were identified before the ETL process.

---

## Data Warehouse Design

A **Star Schema** was designed to support efficient analytical queries.

### Dimension Tables

- **DimDate**
- **DimProduct**
- **DimSeller**
- **DimCustomer**

### Fact Table

- **FactSales**

The **FactSales** table stores sales transactions and references all dimensions through surrogate keys.

The warehouse stores the following analytical measures:

- Product Price
- Freight Cost
- Delivery Days
- Late Delivery Flag

---

## ETL Process

### Extract

Data was extracted from the operational Olist database.

### Transform

The ETL process included the following transformations:

- Generated date keys (`YYYYMMDD`)
- Translated product categories from Portuguese to English
- Standardized city names
- Removed duplicate records
- Mapped business keys to surrogate keys
- Calculated delivery time
- Created late delivery indicators
- Loaded only delivered orders

### Load

The transformed data was loaded into the dimensional warehouse following the Star Schema design.

---

## Data Validation

After loading the warehouse, a comprehensive validation process was performed to guarantee data quality and consistency.

Validation included:

- Source vs. warehouse row count comparison
- Revenue reconciliation
- Order count validation
- Duplicate detection
- Referential integrity verification
- Delivery time validation
- Late delivery validation

The validation confirmed that the ETL pipeline successfully loaded the expected data while maintaining high data quality.
