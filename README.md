Olist Marketplace ETL & Business Intelligence Project
Project Overview

This project demonstrates the complete design and implementation of an ETL (Extract, Transform, Load) pipeline and a Business Intelligence solution using the Olist Brazilian E-commerce Dataset.

The objective was to build a data warehouse capable of answering key business questions about marketplace sales performance, customer behavior, product categories, logistics, and seller performance.

The project covers every stage of the analytics lifecycle:

Data profiling
Data warehouse modeling
Data cleaning and transformation
ETL implementation
Data validation
Business KPI creation
Interactive Power BI dashboard
Business Questions

The project answers the following business questions:

What is the total revenue generated by the marketplace?
How have sales changed over time?
Which product categories generate the highest revenue?
Which product categories receive the most orders?
Which customer states generate the most sales?
Which sellers generate the highest revenue?
What is the average delivery time?
How many orders are delivered late?

Data Profiling

Before building the warehouse, an extensive data profiling process was performed to evaluate the quality of the source data.

The analysis included:

Missing values
Duplicate records
Invalid values
Negative and zero values
Inconsistent text formatting
Leading and trailing spaces
Referential integrity
Outlier detection
Date consistency
Category normalization
Delivery time statistics

This step ensured that potential data quality issues were identified before the ETL process.

Data Warehouse Design

A Star Schema was designed to support analytical queries efficiently.

Dimension Tables
DimDate
DimProduct
DimSeller
DimCustomer
Fact Table
FactSales

The fact table stores sales transactions and references all dimensions through surrogate keys.

The warehouse also stores analytical measures including:

Product price
Freight cost
Delivery time
Late delivery flag
ETL Process
Extract

Data was extracted from the operational Olist database.

Transform

Several transformations were applied during the ETL process, including:

Date key generation (YYYYMMDD)
Translation of product categories from Portuguese to English
Standardization of city names
Duplicate removal
Business key mapping to surrogate keys
Delivery time calculation
Late delivery classification
Filtering only delivered orders
Load

The transformed data was loaded into the dimensional warehouse following the star schema.

Data Validation

A complete validation process was performed after loading the warehouse to guarantee data accuracy.

Validation checks included:

Source vs warehouse row counts
Revenue reconciliation
Order count validation
Duplicate detection
Referential integrity verification
Delivery time validation
Late delivery validation

The validation confirmed that the ETL pipeline successfully loaded the expected data while maintaining data quality and consistency.
